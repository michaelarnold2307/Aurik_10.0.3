# Spec Evidence Reports

Dieser Ordner enthaelt verpflichtende Evidenzberichte fuer direkte main-Commits,
wenn Dateien unter `.github/specs/` geaendert wurden.

Pflicht bei Spec-Aenderung auf main:

- Mindestens eine neue/aktualisierte Markdown-Datei in diesem Ordner
- Enthaltene Pflicht-Tokens:
  - `## Evidenzblock`
  - `Seed`
  - `95 %-CI`
  - `Maintainer Sign-off`

Referenz-Template:

- `docs/guides/SPEC_EVIDENZBLOCK_TEMPLATE.md`

Namensschema (empfohlen):

- `YYYY-MM-DD_<kurzthema>.md`
